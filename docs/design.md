## 4.3 Protótipos das Interfaces

![image](/prototipo_interfaces/inicio.jpg)

1. Análise dos Princípios Gestálticos

Proximidade: A disposição das imagens em categorias como "Temos de tudo", "Personalizado para você" e "Bombando agora" usa proximidade para indicar agrupamentos lógicos. Essa organização ajuda o usuário a reconhecer rapidamente as diferentes seções de eventos, facilitando a navegação.

Semelhança: O uso de caixas de imagem com bordas e uma estética visual consistente (mesmo estilo de imagem e proporção) comunica que todos os itens são eventos e que compartilham a mesma estrutura de conteúdo, o que cria uma sensação de unidade.

Continuidade: O layout em grade cria uma continuidade visual, com as linhas de imagens que guiam o olhar de um lado ao outro da página. Além disso, o design limpo com alinhamento uniforme promove um fluxo natural, facilitando a navegação.

Figura e Fundo: A interface utiliza um fundo preto para destacar as imagens coloridas, fazendo com que os elementos visuais principais se destaquem. Isso cria uma distinção clara entre os eventos (figura) e o fundo escuro da interface.

Simplicidade: O design é simples, com poucos elementos extras, o que mantém o foco nos eventos e suas categorias. A simplicidade ajuda a manter a clareza e evita a sobrecarga visual.

2. Recomendações Ergonômicas

Legibilidade: O contraste entre o texto branco e o fundo escuro melhora a legibilidade, facilitando a leitura das informações dos eventos.

Acessibilidade: A disposição das imagens e o uso de um design em grade ajudam a tornar o conteúdo mais fácil de acessar, além de evitar muita rolagem vertical. A interface parece otimizada para uma navegação confortável, com botões de tamanho adequado para toques em dispositivos móveis.

Conforto Visual: A escolha de cores e o espaçamento adequado entre os elementos ajudam a evitar cansaço visual, tornando a navegação mais confortável.

3. Regras de Design (8 Regras de Ouro)

Consistência: A interface usa elementos visuais consistentes, como o mesmo estilo de imagem e layout, o que facilita a navegação e mantém uma experiência uniforme para o usuário.

Atalhos e Navegação Simples: A presença de uma barra de pesquisa no topo e o menu de navegação no rodapé facilitam o acesso direto a diferentes eventos e opções de navegação, simplificando a interação.

Feedback Imediato: Embora não possamos ver interações reais em uma imagem estática, podemos inferir que elementos como "Acessar" e "Carrinho de compras" seriam interativos e poderiam dar feedback imediato aos usuários.

Minimização da Sobrecarga de Memória: O design agrupa visualmente os eventos, evitando que o usuário precise se lembrar de onde estava ou perder a localização na página.

4. Preparação para Testes com Usuários
Para os testes com usuários na próxima etapa, o protótipo deve incluir funcionalidades interativas, como:

Pesquisa: Testar se a barra de pesquisa permite que os usuários encontrem eventos específicos.
Navegação por Categorias: Verificar se os usuários conseguem explorar as seções de eventos de forma intuitiva.
Interatividade dos Botões: O botão "Acessar" e o carrinho de compras devem simular ações reais para que os usuários entendam seu funcionamento.

![image](/prototipo_interfaces/login.jpg)

1. Análise dos Princípios Gestálticos

Proximidade: A interface agrupa os campos de entrada de email e senha, além do botão "Entrar", de forma próxima e organizada, criando uma seção clara para o usuário saber onde inserir suas informações para fazer login.

Semelhança: Os campos de email e senha têm o mesmo estilo visual (caixas de texto com bordas e fonte semelhante), reforçando que ambos têm a mesma função de entrada de dados.

Figura e Fundo: O fundo escuro ao redor do formulário destaca a seção de login, fazendo com que a atenção do usuário se volte diretamente para a caixa de login, que é o foco da página.

Continuidade: O alinhamento vertical dos elementos guia o usuário do topo para baixo, seguindo uma ordem lógica (primeiro o email, depois a senha, e em seguida o botão de login), o que facilita o fluxo de interação.

Simplicidade: A interface é minimalista e foca exclusivamente na ação de login, sem elementos visuais adicionais. Isso torna a interação simples e direta.

2. Recomendações Ergonômicas

Legibilidade: A escolha de um texto branco sobre fundo escuro aumenta a legibilidade. O tamanho das fontes e o espaçamento entre os campos e o botão "Entrar" são adequados para facilitar a leitura e o uso.

Acessibilidade: Os campos e o botão "Entrar" são grandes o suficiente para que possam ser facilmente tocados em dispositivos móveis. Além disso, o link “Registrar-se” oferece uma alternativa para quem não tem conta, aumentando a usabilidade.

Conforto Visual: A simplicidade e o contraste ajudam a reduzir o cansaço visual, especialmente em uma tela de login, que é normalmente usada de forma rápida.

3. Regras de Design (8 Regras de Ouro)

Consistência: O layout é simples e uniforme. A cor roxa é usada tanto no título ("Acessar") quanto no botão "Entrar" e no link "Registrar-se", reforçando a identidade visual da interface.

Feedback Imediato: Embora o feedback imediato não seja visível em uma imagem estática, é esperado que o botão "Entrar" dê uma resposta visual, como uma mudança de cor ao ser clicado, indicando que a ação foi registrada.

Minimização da Sobrecarga de Memória: A interface é intuitiva e fácil de entender. Como a tela é simples, o usuário não precisa memorizar nenhuma informação complexa para completar a tarefa de login.

Controle pelo Usuário e Liberdade: A opção de “Registrar-se” permite que o usuário mude de ação caso perceba que ainda não possui uma conta.

4. Preparação para Testes com Usuários

Para os testes com usuários, as funcionalidades que devem ser interativas no protótipo incluem:

Campo de Login e Senha: Verificar se os campos aceitam a entrada de dados de forma fácil e se indicam possíveis erros, como email inválido.

Botão Entrar: Simular o processo de login e confirmar que o botão responde visualmente ao clique.

Link Registrar-se: Testar se o link “Registrar-se” redireciona o usuário para a tela de cadastro de maneira intuitiva.

![image](/prototipo_interfaces/registro.jpg)

1. Análise dos Princípios Gestálticos

Proximidade: A interface agrupa todos os campos de entrada de dados (email, nome, senha e confirmação de senha), além do botão “Registrar-se”, em um bloco coeso. Isso facilita a navegação, indicando que todos os campos pertencem ao processo de criação de conta.

Semelhança: Todos os campos de entrada têm o mesmo estilo visual (caixas de texto com bordas e fonte semelhante), reforçando que todos fazem parte de uma mesma categoria de interação (entrada de dados). O botão “Registrar-se” e o título “Registrar-se” compartilham uma mesma cor, o que também reforça a identidade visual e funcionalidade.

Figura e Fundo: O fundo escuro ao redor do formulário destaca a seção de cadastro, direcionando a atenção do usuário para o conteúdo principal e criando um contraste que facilita a visualização.

Continuidade: O alinhamento vertical dos elementos (email, nome, senha, confirmar senha, e o botão de registro) segue um fluxo lógico de cima para baixo, o que orienta o usuário na ordem de preenchimento dos campos.

Simplicidade: A interface mantém-se minimalista e foca exclusivamente no objetivo de registrar um novo usuário, sem distrações.

2. Recomendações Ergonômicas

Legibilidade: A interface usa texto branco sobre fundo escuro, o que melhora a legibilidade. Os campos de entrada de dados e o botão estão bem espaçados, evitando sobrecarga visual.

Acessibilidade: O tamanho dos campos e do botão “Registrar-se” é adequado para facilitar o uso tanto em desktops quanto em dispositivos móveis, garantindo que sejam facilmente clicáveis.

Confirmação Visual: A presença do campo “Confirmar Senha” ajuda a minimizar erros de digitação e, assim, aumenta a segurança e a confiança do usuário no cadastro.

3. Regras de Design (8 Regras de Ouro)

Consistência: O layout é simples e uniforme, e o uso da cor roxa no título e no botão reforça a identidade visual. Além disso, a interface segue o mesmo padrão visual do Protótipo 2, criando uma experiência consistente para o usuário.

Feedback Imediato: Como é um protótipo estático, o feedback não pode ser demonstrado aqui, mas espera-se que o botão "Registrar-se" forneça uma resposta visual (como uma mudança de cor) quando clicado, indicando que a ação foi registrada.

Minimização da Sobrecarga de Memória: A interface é simples e autoexplicativa, reduzindo a necessidade de o usuário lembrar-se de informações complexas para completar o processo de registro.

Controle pelo Usuário e Liberdade: O link “Acessar” permite que o usuário volte à tela de login caso perceba que já tem uma conta, promovendo flexibilidade na navegação.

4. Preparação para Testes com Usuários

Campos de Registro: Verificar se os campos de email, nome, senha e confirmação de senha aceitam a entrada de dados corretamente. Testar se o campo "Confirmar Senha" reconhece discrepâncias com o campo "Senha" e exibe uma mensagem de erro, se aplicável.

Botão Registrar-se: Simular o processo de registro e confirmar que o botão responde visualmente ao clique.

Link Acessar: Testar se o link “Acessar” redireciona o usuário para a tela de login de maneira intuitiva.
